# 用户画像模板

用户研究员在构建用户画像时应使用以下结构。

## 画像结构

```markdown
### 用户画像: [名称]

**基本信息**
- 年龄段: [如 25-35 岁]
- 职业: [如 程序员/学生/白领]
- 技术水平: [如 高/中/低]
- 使用场景: [如 通勤/睡前/工作间隙]

**核心需求**
- [需求 1] — 优先级: 高/中/低
- [需求 2] — 优先级: 高/中/低
- [需求 3] — 优先级: 高/中/低

**当前痛点**
- [痛点 1] — 严重程度: 高/中/低
- [痛点 2] — 严重程度: 高/中/低

**行为特征**
- 使用频率: [如 每天/每周/偶尔]
- 单次时长: [如 10分钟/1小时]
- 关键路径: [如 打开→搜索→阅读→关闭]

**决策因素**
- 选择产品的理由: [如 内容多/体验好/朋友推荐]
- 流失的原因: [如 广告太多/加载慢/找不到想看的]
```

## 需求优先级矩阵

| 需求 | 硬需求/软需求 | 当前满足度 | 改进成本 | 价值 |
|------|--------------|-----------|---------|------|
| ... | 硬/软 | 0%/50%/100% | 高/中/低 | 高/中/低 |

## 场景描述规范

每个场景必须包含：
1. **触发条件** — 用户在什么情况下进入这个场景
2. **用户目标** — 用户想达到什么结果
3. **当前路径** — 用户现在怎么做（步骤 1→2→3）
4. **痛点** — 哪一步让用户不爽
5. **改进路径** — 方案如何优化这个流程
