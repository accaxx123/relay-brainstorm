# 技术分析检查清单

技术专家在分析时应覆盖以下维度。

## 可行性检查

- [ ] 核心功能是否技术可行
- [ ] 是否有阻断性技术难题
- [ ] 现有技术栈是否支持
- [ ] 是否需要引入新技术

## 工作量估算

- [ ] MVP 最小范围定义清楚
- [ ] 每个功能点有独立估算
- [ ] 包含测试和联调时间
- [ ] 包含部署和上线时间

## 技术选型

- [ ] 推荐了具体的技术栈/工具
- [ ] 说明了选型理由
- [ ] 考虑了备选方案
- [ ] 评估了学习成本

## 风险评估

- [ ] 识别了技术债
- [ ] 识别了性能瓶颈
- [ ] 识别了安全隐患
- [ ] 每个风险有缓解方案

## 分阶段

- [ ] 区分了 MVP / 完善版 / 优化版
- [ ] 每个阶段有明确的交付物
- [ ] 阶段之间有清晰的依赖关系
